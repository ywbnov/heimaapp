# 自定义分类与分类管理设计

## 目标

在现有两级分类模型上增加用户自定义分类能力：用户可以新增一级分类或二级分类，可以重命名、启用/停用和逻辑删除自定义分类；预置分类保持只读，不允许被修改或删除。

## 约束与行为

- 分类继续使用两级父子结构。
- 新增一级分类时 `parentId` 为空；新增二级分类时必须选择一个一级分类。
- 每个分类增加 `isBuiltin` 标记。默认目录中的分类为预置分类，用户新增分类为自定义分类。
- 每个分类增加 `isDeleted` 标记。逻辑删除只允许自定义分类，并同时将 `enabled` 设为 false；数据库行和历史账目都保留。
- 预置分类不允许重命名、启用/停用或删除。预置一级分类仍可作为自定义二级分类的父级。
- 自定义分类允许重命名、启用/停用和逻辑删除。已删除分类不能再次用于新账目。
- 记账选择器只展示启用且未删除的分类；历史账目和统计仍能读取已删除分类的名称与层级。

## 数据与迁移

`categories` 表增加 `is_builtin INTEGER NOT NULL DEFAULT 1` 和 `is_deleted INTEGER NOT NULL DEFAULT 0`。打开已有数据库时通过迁移补充缺失字段，现有默认分类视为预置分类。新增分类使用稳定的随机 ID，保留 `sort_order` 以维持现有排序规则。

账目表继续保存分类 ID 和名称快照，不修改历史账目。备份恢复格式增加两个分类字段，同时兼容缺少字段的旧版本备份：已知默认分类按当前内置目录恢复，未知分类按自定义分类恢复。

## API 与权限边界

主进程仓库新增 `create`、`rename`、`softDelete` 方法，并在仓库层拒绝对预置分类的变更。IPC/preload 暴露对应的创建、重命名和删除操作；输入需校验名称非空、父级存在且为一级分类。已有 `setEnabled` 也必须拒绝预置分类。

## 界面

设置页新增分类弹窗，允许填写名称并选择“一级分类”或一个现存一级分类作为父级。列表显示预置/自定义状态；自定义分类提供编辑、启用/停用和逻辑删除入口，预置分类只读。删除前显示确认提示。

## 测试

- 数据库集成测试覆盖新增一级/二级分类、重命名自定义分类、逻辑删除、历史账目保留，以及所有预置分类保护。
- IPC 契约测试覆盖新增、重命名、删除参数校验和预置分类拒绝。
- 分类规则/表单测试覆盖已删除分类不可用于新账目。
- 设置页渲染测试覆盖自定义分类操作入口和预置分类只读状态。
